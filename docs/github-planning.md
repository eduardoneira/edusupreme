# GitHub Planning

This repo uses GitHub Issues and Milestones as the lightweight planning system for the table tennis tournament platform.

## Milestones

- `M0 - Foundation`
- `M1 - First Vertical Slice`
- `M2 - Angular App Experience`
- `M3 - Security`
- `M4 - Tournament Domain`
- `M5 - Microservices Split`
- `M6 - Kafka & Events`
- `M7 - Cloud Readiness`

## Issue Format

Every implementation task should stay small and use this structure:

```md
## Goal

## Acceptance Criteria

## Technology Practiced

## Notes / Resources
```

## First Roadmap Slice

Create detailed issues only for `M0 - Foundation` and `M1 - First Vertical Slice` at first. Later milestones stay as roadmap markers until the previous milestone is mostly complete.
